# Plotline Glossary

Key terms and concepts used throughout the documentation.

---

## Video & Audio

| Term | Definition |
|------|------------|
| **Video** | Moving image footage recorded by your, camera, or phone. Plotline works with video files (MOV, MP4, MXF, etc.). |
| **Audio** | Sound track extracted from video for processed. Plotline extracts audio as 16kHz mono WAV files for transcription. |
| **Transcript** | Text representation of speech, generated by Whisper. Includes timestamps and speaker labels (if diarization is enabled). |
| **EDL** | Edit Decision List. A universal timeline format for DaVinci Resolve, Premiere Pro. and other NLEs. See [Export Guide](export-guide.md) for details. |
| **FCPXML** | Final Cut Pro XML, A timeline format for Final Cut Pro with markers, keywords, and metadata. See [Export Guide](export-guide.md#fcpxml) for details. |
| **NLE** | Non-Linear Editor. DaVinci Resolve, Premiere Pro, Final Cut Pro, Avid Media Composer. |

| **Handle** | Extra padding (in frames) added before and after each clip in the exported timeline. Gives editors room for transitions. Default: 12 frames (~0.5s at 24fps). |

## AI & LLM

| Term | Definition |
|------|------------|
| **LLM** | Large Language Model. AI that can understand text and generate narrative. Ollama, LM Studio, Claude, or OpenAI GPT-4 can run these models locally. |
| **Ollama** | Tool for running LLMs locally on your machine. Handles model download, serving, and inference. |
| **Diarization** | The process of identifying and labeling different speakers in an audio recording. Powered by pyannote.audio. |
| **Speaker** | A unique voice identified by diarization. Labeled as SPEAKER_00, SPEAKER_01, etc. by default. |
| **Role** | Classification of a speaker: `interviewer` (asks questions), `subject` (being interviewed), or `unknown`. |
| **include_in_edl** | Whether a speaker's segments should appear in the exported timeline. Typically set to `false` for interviewers. |

## Analysis Concepts

| Term | Definition |
|------|------------|
| **Whisper** | Open-source speech recognition model by OpenAI. Transcription performed locally. Language is auto-detected by Whisper and carried through the entire pipeline. |
| **Delivery Analysis** | Measuring vocal characteristics (energy, pace, pitch, pauses) to score how engaging each segment sounds. |
| **Enrichment** | Merging transcript data with delivery analysis into unified segment records. |
| **Theme Extraction** | LLM analysis to identify recurring topics and ideas across interviews. |
| **Synthesis** | Cross-interview analysis to unify themes and find the best takes for each topic. |
| **Narrative Arc** | The structured storyline (Hook → Body → Climax → Resolution) built by the LLM from selected segments. |
| **Cultural Flags** | Segments that may require community or cultural review before publication (sacred references, sensitive terminology). Enabled by `cultural_flags: true` in config or with `commercial-doc` profile. |

## Export & Timeline

| Term | Definition |
|------|------------|
| **Handle** | Extra padding (in frames) added before and after each clip in the exported timeline. Gives editors room for transitions. Default: 12 frames (~0.5s at 24fps). |
| **Smart Handles** | Automatic handle reduction when natural pauses are too short for the default handle size to which would cut into the next speaker's dialogue. |
| **Alternates** | Secondary timeline containing all alternate takes for a given segment. allowing easy comparison and swapping in your NLE. |
